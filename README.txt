Cloud Cartel – Static Showcase (ohne Shop)

STRUKTUR
- index.html
- /img/   (Bilder hier ablegen, z.B. webp/jpg; EXIF vorher löschen)

DEPLOY (GitHub Pages, schnell & ohne Tracking)
1) Neue E-Mail (z. B. Proton) nur für Cloud Cartel verwenden.
2) GitHub-Account mit dieser Mail anlegen (2FA aktivieren).
3) Neues Repo: cloud-cartel-site  → Public.
4) Datei index.html hochladen (Drag & Drop). Ordner /img/ anlegen und Bilder hochladen.
5) Settings → Pages → Source = Deploy from a branch → main / root → Save.
6) Nach kurzer Zeit live unter: https://<dein-user>.github.io/cloud-cartel-site/

BENUTZERDEFINIERTE DOMAIN (optional)
- Domain mit WHOIS-Privacy registrieren.
- In GitHub Pages unter Custom domain z. B. www.deinedomain.tld eintragen (legt CNAME an).
- Beim Domain-Registrar DNS: CNAME www → <dein-user>.github.io
- Apex-Domain auf www weiterleiten (Registrar-Redirect).

BILDER
- Eigenes Logo/Bilder in /img/ speichern und die Pfade in STRAINS im index.html anpassen.
- EXIF/Geodaten vorher entfernen (z.B. exiftool -all= *.jpg).

RECHT & PRIVACY
- Kein Shop, keine Cookies, kein externes Tracking.
- In Deutschland meist Impressumspflicht: rechtlich prüfen.
- Alters-Hinweis vorhanden, ersetzt keine echte Altersprüfung.
